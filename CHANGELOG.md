# 변경 이력 — 인천광역시 1인가구·세대구조 대시보드

최초 기획부터 지금까지 무엇을, 왜 바꿨는지 시계열로 정리한 기록입니다.
(정리일: 2026-07-20 · 대상: 인천 10개 시군구 · 정적 · 단일 파일 배포)

---

## 0. 기획 — 무엇을 만들지 정함
- 인구/세대 데이터를 **인천광역시**로 한정해 대시보드로 시각화하기로 함.
- **API로 받아오는 구조는 피하고**, 공식 기관에서 **파일(CSV·SHP)로 받아** 처리.
- **React + Vite**, **서버 배포 없음**, 작업환경 macOS.
- 처음엔 글라스모피즘을 원했으나 이후 방향이 계속 조정됨(아래 참조).

## 1. 초기 구축 — 데이터 파이프라인 + 대시보드 골격
- `extract.py` 작성: KOSIS 가구(DT_1JC1511·2023), 행정안전부 주민등록(2025 월별), 국토지리정보원 경계 SHP를 정제·집계해 정적 JSON 생성.
- **왜 이렇게**: 데이터가 작아 빌드 전에 한 번만 전처리하면 됨 → 서버·API 불필요.
- 경계는 **GDAL 없이**(pyshp·pyproj·shapely) 변환 — macOS에서 설치가 쉬움.
- React 대시보드 초안 + `vite-plugin-singlefile`로 **단일 `index.html`** 산출(React 몰라도 파일 하나로 열림).

## 2. 지도 중심으로 전면 개편
- 타일 격자 형태의 어설픈 지도를 걷어내고 **Leaflet + CARTO(OSM)** 실제 지도로 교체.
- 경계를 **읍면동 → 시군구로 병합(dissolve)** — 분석은 구 단위이므로 읍면동 경계는 표출하지 않음.
- **양방향 인터랙션**: 지도 클릭 → 상세·차트 갱신 / 순위·산점도 클릭 → 지도 반응 / 지표·필터 → 지도 채색.

## 3. 디자인 톤을 기존 AR6 프로젝트에 맞춤
- 색상·구조를 사용자의 이전 프로젝트(기후정책 AR6)와 통일(파랑 계열, Pretendard 톤, 사이드바/카드 레이아웃).
- 불필요하게 만들었던 랜딩 페이지 제거 — 요구는 색상·데이터 선택부의 정렬이었음.

## 4. AI가 지어낸 내용 제거
- '핵심 진단', '정책 시사점' 같은 **AI 생성 해석 문구 제거**.
- AI가 임의로 만든 **1인가구 유형 분류(청년형/고령형/균형형) 제거** → 데이터 기반 '인천 평균 대비' 필터로 대체.
- 이후 데이터(`oneType`)에 남아 있던 잔재까지 완전 삭제.

## 5. 가독성·디테일 정리
- 시군구 클릭 시 생기던 사각 포커스 테두리 제거.
- **범례는 축소(개요) 상태에서만** 표시 → 확대 시 화면 정리.
- 산점도(④) 글씨·원 크기 축소로 다른 카드와 밀도 맞춤. 헤더의 홍보성 문구 제거.

## 6. 격자 밀집도 히트맵 추가
- 구를 확대하면 **500m 격자 밀집도 히트맵**이 겹쳐 표시(구 안에서 몰린 위치 확인).
- 사이드바에 **격자 토글** 추가. 확대 시 시군구는 외곽선만 남기고 격자가 주가 되게 함.
- **왜 표본인가**: 실제 SGIS 격자 파일이 없어 시군구 경계·1인가구 수로 만든 표본. `data/raw/grid`에 실제 파일을 넣으면 실제값으로 대체(범례에 '표본' 표기).

## 7. 지도를 1차 정보면으로
- 각 시군구 위에 **선택 지표값을 상시 라벨**로 표시(색만이 아니라 수치를 지도에서 바로 읽음). 확대(격자) 시 라벨은 숨김.

## 8. 프론트엔드 실시간 연산 기능
- **가중 종합지수 슬라이더**: 여러 지표를 정규화(min-max)해 슬라이더 가중치로 가중 평균 → 0~100점을 **브라우저에서 실시간 계산** → 지도·순위·평균 즉시 갱신.
- **산점도 상관계수·회귀선**: 피어슨 상관계수(r)와 최소제곱 회귀선을 **브라우저에서 계산**해 표시.
- **왜**: "프론트에서 실제로 연산이 도는가"를 보여주기 위해. 정적 표시가 아니라 입력에 반응해 재계산.

## 9. UI 수정 (사용성)
- 지도 지표를 **아코디언**으로: 선택한 항목만 펼치고 나머진 접힘, 다시 누르면 접힘, 헤더로 전체 접기.
- 좌측 사이드바 **폭 확대**(246→292px), "브라우저 실시간 계산" 배지·지표명 **줄바꿈 해결**.
- 지도 점수 라벨을 **가장 큰 폴리곤 중심**에 찍도록 수정 → 연수구가 바다에·옹진군이 빈 구석에 찍히던 문제 해결.

## 10. 정보구조·인터랙션 개선 (현재)
- **데이터 출처 모달**: 헤더의 'ⓘ 데이터 출처·시점·구조' 버튼 → 4개 데이터셋(가구·세대·경계·SGIS 격자)의 출처·시점·구조·활용을 정리(뒤에 추가한 격자까지 반영).
- **사이드바 기능 그룹화**: '지도에 반영(색·강조·격자)'과 '표·상세에 반영(선택 지역)'을 섹션으로 분리 — 기능이 중구난방이지 않게.
- **패널 접기 + 줌 분리 + 오프셋**:
  - 중앙 상세 패널을 접어 지도를 넓게 볼 수 있음.
  - **패널에서 지역을 골라도 자동 줌하지 않음**(강조만). 확대는 **지도를 직접 클릭할 때만**.
  - 확대·개요 시 **좌측 패널이 가리는 폭만큼 여백을 보정** → 선택한 구가 패널 뒤로 숨지 않고 보이는 지도 영역 중앙에 옴.
- **지수 이름·방향 의미화**: '종합지수' → **'1인가구 집중지수'**. 높을수록 1인·고령·소규모 가구 집중을 뜻하도록 방향을 맞추고(세대당 인구는 작을수록 가점) 이름부터 그 의미가 드러나게 함.
